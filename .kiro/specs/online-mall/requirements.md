# 在线商城需求文档

## 介绍

开发一个现代化的在线商城web应用，使用Vue3+Vite技术栈，面向年轻用户群体，提供充满活力的购物体验。

## 术语表

- **商城系统**: 整个在线购物平台
- **用户**: 使用商城进行购物的消费者
- **商品**: 在商城中销售的产品
- **购物车**: 用户临时存储待购买商品的容器
- **订单**: 用户确认购买的商品清单和交易记录
- **访问令牌**: 用于验证用户身份的JWT令牌
- **认证系统**: 负责用户登录、注册和权限管理的子系统
- **HTTP客户端**: 封装的axios实例，用于统一管理API请求
- **轮播图**: 首页顶部的图片轮播展示组件

## 需求

### 需求 1: 商城首页展示

**用户故事:** 作为用户，我希望看到一个吸引人的商城首页，以便快速了解商城的商品和优惠信息。

#### 验收标准

1. 当用户访问商城首页时，系统应显示导航栏、轮播图、商品分类和推荐商品
2. 当用户查看首页时，系统应展示年轻活力的设计风格，包括鲜艳的色彩和现代化的布局
3. 当用户在首页浏览时，系统应提供响应式设计，适配不同屏幕尺寸
4. 当用户点击导航菜单时，系统应提供平滑的页面跳转体验
5. 当首页加载时，系统应在3秒内完成所有内容的渲染

### 需求 2: 商品展示和分类

**用户故事:** 作为用户，我希望能够浏览不同分类的商品，以便找到我感兴趣的产品。

#### 验收标准

1. 当用户点击商品分类时，系统应显示该分类下的所有商品
2. 当用户查看商品列表时，系统应显示商品图片、名称、价格和评分
3. 当用户点击商品时，系统应跳转到商品详情页面
4. 当商品列表加载时，系统应支持分页或无限滚动加载
5. 当用户搜索商品时，系统应根据关键词筛选相关商品

### 需求 3: 购物车功能

**用户故事:** 作为用户，我希望能够将商品添加到购物车，以便统一结算购买。

#### 验收标准

1. 当用户点击"加入购物车"按钮时，系统应将商品添加到购物车并更新购物车数量
2. 当用户查看购物车时，系统应显示所有已添加的商品、数量、单价和总价
3. 当用户修改购物车商品数量时，系统应实时更新总价
4. 当用户删除购物车商品时，系统应从购物车中移除该商品
5. 当用户清空购物车时，系统应移除所有商品并重置购物车状态

### 需求 4: 用户界面和交互

**用户故事:** 作为用户，我希望界面美观易用，以便获得良好的购物体验。

#### 验收标准

1. 当用户使用商城时，系统应提供年轻活力的视觉设计，包括渐变色彩和动画效果
2. 当用户进行操作时，系统应提供即时的视觉反馈和加载状态
3. 当用户在移动设备上访问时，系统应提供优化的触摸交互体验
4. 当用户操作出错时，系统应显示友好的错误提示信息
5. 当页面切换时，系统应提供平滑的过渡动画

### 需求 5: 路由和状态管理

**用户故事:** 作为开发者，我希望应用有清晰的路由结构和状态管理，以便维护和扩展功能。

#### 验收标准

1. 当用户访问不同页面时，系统应通过Vue Router管理页面路由
2. 当应用状态发生变化时，系统应通过状态管理工具（如Pinia）统一管理
3. 当用户刷新页面时，系统应保持当前页面状态和购物车数据
4. 当路由切换时，系统应正确加载对应的页面组件
5. 当应用初始化时，系统应从本地存储恢复用户的购物车数据

### 需求 6: 性能和用户体验

**用户故事:** 作为用户，我希望商城加载快速且运行流畅，以便获得优质的购物体验。

#### 验收标准

1. 当用户首次访问时，系统应在3秒内完成首页加载
2. 当用户切换页面时，系统应在1秒内完成页面渲染
3. 当用户滚动页面时，系统应保持60fps的流畅度
4. 当图片加载时，系统应提供占位符和渐进式加载
5. 当网络较慢时，系统应提供离线缓存和优雅降级

### 需求 7: 用户认证和授权

**用户故事:** 作为用户，我希望能够注册账号并登录系统，以便使用个性化的购物功能。

#### 验收标准

1. 当用户访问登录页面时，系统应显示用户名/邮箱输入框、密码输入框和登录按钮
2. 当用户输入有效的用户名或邮箱和密码并点击登录时，系统应调用登录接口并返回访问令牌和用户信息
3. 当登录成功时，系统应将访问令牌、用户信息和权限信息存储到状态管理和本地存储中
4. 当用户点击注册链接时，系统应跳转到注册页面
5. 当用户在注册页面填写必填信息（用户名、邮箱、邮箱验证码、密码）时，系统应验证输入格式的有效性
6. 当用户请求邮箱验证码时，系统应发送验证码到指定邮箱，并在60秒内禁止重复发送
7. 当用户提交注册表单时，系统应验证两次密码输入一致性
8. 当注册成功时，系统应显示成功提示并在2-3秒后自动跳转到首页
9. 当登录或注册页面加载时，系统应预留Google登录按钮位置
10. 当用户登录状态过期时，系统应提示用户重新登录

### 需求 8: HTTP请求管理

**用户故事:** 作为开发者，我希望有统一的HTTP请求封装，以便简化API调用和错误处理。

#### 验收标准

1. 当发起HTTP请求时，系统应提供getAction、postAction、putAction、deleteAction等封装方法
2. 当发送请求时，系统应在请求拦截器中自动添加"Bearer "前缀的访问令牌到Authorization请求头
3. 当接收响应时，系统应在响应拦截器中检查是否存在"new-token"响应头
4. 当响应包含"new-token"时，系统应自动更新本地存储的访问令牌
5. 当请求失败时，系统应统一处理错误并返回标准化的错误信息
6. 当响应数据格式为标准封装格式时，系统应自动提取result字段中的实际数据
7. 当发生401未授权错误时，系统应清除用户登录状态并跳转到登录页面

### 需求 9: 动态内容管理

**用户故事:** 作为用户，我希望看到从后端获取的动态轮播图内容，以便了解最新的促销和推荐商品。

#### 验收标准

1. 当首页加载时，系统应从后端接口获取轮播图数据
2. 当轮播图数据包含展示图时，系统应优先使用展示图（displayImg）
3. 当轮播图数据不包含展示图时，系统应使用主图（mainImg）作为展示图片
4. 当轮播图加载时，系统应显示加载占位符
5. 当轮播图加载失败时，系统应显示默认占位图或错误提示
6. 当轮播图展示时，系统应按sortOrder字段排序显示
7. 当轮播图设计时，系统应符合现代审美标准，包括适当的图片比例、圆角和阴影效果
